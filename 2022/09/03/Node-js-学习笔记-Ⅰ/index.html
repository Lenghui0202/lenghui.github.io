Node.js简介

1.Node.js是一个能够在服务器端运行JS的开放源代码、跨平台JS运行环境

2.Node采用Google开发的V8引擎运行JS代码，使用事件驱动、非阻塞和异步I/O模型等技术来提高性能，可以优化应用程序的传输量和规模

3.Node大部分基本模块都用JS编写。在Node出现之前，JS通常作为客户端程序设计语言使用，以JS写出的程序常在用户的浏览器上运行

4.目前，Node已被IBM、Microsoft、Yahuo！、Walmart等等很多企业采用

fs文件系统模块

什么是fs文件系统模块？

fs模块是Node.js官方提供的，用来操作文件的模块。他提供了一系列的方法和属性，用来满足用户对文件的操作需求

导入fs模块的方法如下

const fs = require('fs')

fs.readFile() —— 读取指定文件中的内容

fs.readFile()方法可以读取文件中的内容，语法格式如下：

fs.readFile(path[,options],callback)

解读：

path为必选参数，字符串，表示文件的路径

options为可选参数，表示以什么编码格式来读取文件

callback为必选参数，文件读取完成后，通过回调函数拿到读取的结果

实例代码

// 导入fs模块

const fs = require('fs')

// 调用fs.readFile()方法读取文件

fs.readFile("./files/1.txt","utf-8",function(err,dataStr){

// 打印失败的结果

// 如果成功，err则为null；如果失败，则为错误对象，dataStr 则为 underfined

console.log(err)

// 分割线

console.log("-------------")

// 打印成功的结果

console.log(dataStr)

})

运行结果截图

根据以上内容，我们便可以判断fs.readFile()方法读取文件是否成功

fs.writeFile() —— 向指定的文件中写入内容

fs.writeFile()方法可以向指定的文件中写入内容，语法格式如下：

fs.writeFile(file,data[,options],callback)

解读：

file为必选参数，需要指定一个文件路径的字符串，表示文件的存放路径

data为必选参数，表示写入的内容

options为可选参数，表示写入文件内容的格式，默认值为utf-8

callback为必选参数，表示写入完成后的回调函数

// 导入fs模块

const fs = require('fs')

// 调用fs.writeFile()方法读取文件

fs.writeFile('./files/1.txt','abcd','utf-8',function(err){

// 写入失败的结果 成功 err 则为 null ；失败则为错误对象

console.log(err)

})

注意：此方法会覆盖原来的内容，同时，此方法只能用来创建文件，不能用来创建路径

__dirname —— 路径动态拼接问题

在使用fs模块操作时，如果提供的操作路径是以 ./ 或者 ../ 开头的相对路径时，很容易出现路径动态拼接错误的问题。

原因：代码在运行的时候，会以执行node命令时所处的目录，动态拼接出被操作文件的完整路径。

例如：您的代码如下：

const fs =require('fs')

fs.readFile('./files/1.txt','utf8',function (err,dataStr) {

if (err) {

​ return console.log('文件读取失败!' + err)

}

console.log('读取文件成功！' + dataStr)

})

其中，1.txt所处路径如下：F:\01-Study\NodeJs\day01\files\1.txt

而代码所处文件的路径为：F:\01-Study\NodeJs\day01\04fs路径拼接.js

如果您如同以下运行，则会成功：

但当您像以下方式运行时，则会报错：

您会发现，此路径F:\01-Study\NodeJs\files\1.txt并不是该文件的路径，这时便可以用以下方法解决。

解决方案：在使用fs模块操作文件时，直接提供完整的路径，不要提供 ./ 或者 ../ 开头的相对路径，从而防止动态拼接的问题，但是这种方法会导致代码的移植性非常差，而且不利于维护。因此，我们可以使用 __dirname，所以以后的代码，均使用此方法。__dirname 表示当前文件所处的目录

代码案例：

const fs =require(‘fs’)

fs.readFile(__dirname + '/files/1.txt','utf8',function (err,dataStr) {

if (err) {

​ return console.log('文件读取失败!' + err)

}

console.log('读取文件成功！' + dataStr)

})

path路径模块

path路径模块是Node.js官方提供的，用来处理路径的模块。他提供了一系列的方法和属性，用来满足用户对路径的操作需求

导入模块的方法如下

const path = require('path')

path.join() —— 路径拼接

path.join()方法可以把多个路径片段拼接为完整的路径字符串

代码案例：

const path = require('path')

const pathStr = path.join('/a','/b/c','../','./d','e')

console.log(pathStr)

注意：其中的 ../ 会抵消前面的路径

path.basename() —— 获取路径中的文件名

使用path.basename() 方法可以从一个文件路径中，获取到文件的名称部分

代码案例：

// 引入path模块

const path = require('path')

// 文件的存放路径

const fpath = '/files/1.txt'

// 输出完整的名称（包括文件格式）

const fullName = path.basename(fpath)

console.log(fullName)

// 只输出文件名

const nameWithoutExt = path.basename(fpath,'.txt')

console.log(nameWithoutExt)

模块化

模块化的基本概念

模块化是指解决一个复杂的问题时，自顶向下逐层把系统划分成若干模块的过程。对于整个系统来讲，模块是可组合，分解和更换的单元。

编程领域中的模块化，就是遵守固定的规则，把一个大文件拆成独立并互相依赖的多个小模块。把代码进行模块化拆分的好处：1.提高了代码的复用性 2. 提高了代码的可维护性 3. 可以实现按需加载

模块化的规范

Node.js 遵循了CommonJS 模块化规范，CommonJS 规定了模块的特性和各模块之间如何相互依赖

CommonJS 规定：

  1. 每个模块内部，module 变量代表当前模块
  2. module 变量是一个对象，它的exports （module.exports）属性是对外的接口
  3. 加载某个模块，其实是加载该模块的 module.exports 属性。 require() 方法用于加载模块

Node.js中模块化

Node.js中根据模块来源的不同，将模块分为了3大类，分别是：

· 内置模块(内之模块是官方提供的，例如fs，path，http等)

· 自定义模块(用户创建的每个.js文件，都是自定义模块)

· 第三方模块(由第三方开发出来的模块，非Node.js官方提供的，也不是用户自定义的模块，使用前需要先下载)

**require()**方法加载模块

使用**require()**方法，可以加载需要的模块进行使用。

// 加载内置模块

const fs = require('fs')

// 加载用户的自定义模块(填入路径)

const custom = require('./custom.js')

// 加载第三方模块

const moment = require('moment')

注：使用此方法会执行被加载模块中的代码，同时可以省略后缀名

模块作用域

和函数作用域类似，在自定义模块中定义的变量，方法等成员，只能在当前模块内被访问，这种模块级别的访问限制，叫做模块作用域

好处：防止了全局变量污染的问题
